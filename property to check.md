## بخش ۲: سرور شرکت (Production)
==================================================
- اسم سرور (در SSMS): A-SHAREPOINT-T
- IP سرور: 172.31.15.196
- در رشته‌ی اتصال از "." استفاده شده (چون اپ روی همان سرور دیتابیس اجرا می‌شود)
- روش اتصال: SQL Authentication
- یوزر: actionplan  /  پسورد: Aa@123456
- اسم دیتابیس: ActionPlanDB
- فعلاً اپ و دیتابیس روی یک سرور هستند
- فایل: appsettings.Production.json

### اگر بعداً دیتابیس از اپ جدا شد:
در appsettings.Production.json به‌جای "." اسم/IP سرور دیتابیس را بگذار:
Server=172.31.15.196  یا  Server=A-SHAREPOINT-T

### اجرای اپ روی سرور: IIS یا Windows Service
چون SQL Auth استفاده می‌کنیم، دیگر دردسر حساب سرویس نداریم (یوزر/پسورد در رشته است).

==================================================
